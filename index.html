springboot零基础入门

Spring Boot是由Pivotal团队提供的全新框架，Spring Framework也是他们的杰作，Spring最早是由Rod Johnson和Pivotal团队开发的。在开发Spring Framework之前还著书《Expert One-on-One J2EE Development without EJB》用于批判当时业界主流后端框架EJB，缘由是EJB过于臃肿，这本书为Spring Framework的诞生奠定了理论基础。由于Rod Johnson在Spring Framework的理论和实现上的卓出贡献，其被称为Spring之父。—为大佬献上狗头

Spring Boot + Mybatis实现xml版本的增删查改demo

​ 以下代码可以参考我的github https://github.com/lonelyProgramMonkey/SpringBoot-demos/tree/master/springboot_mybatis_xml ,如果觉得本文对你有帮助的话，还请star一下我的github，你的支持就是对我最大的鼓励。

​ 我们首先从增删查改的demo来入门，hello world就不赘述了。首先你需要配置一个数据库进行数据的增删查改。这里我推荐如今的主流数据库MySQL，具体可以参考菜鸟教程https://www.runoob.com/mysql/mysql-tutorial.html 进行安装。

​ 安装MySQL后导入users.sql https://github.com/lonelyProgramMonkey/SpringBoot-demos/blob/master/springboot_mybatis_xml/users.sql 到MySQL的test数据库来建表，具体可以使用MySQL命令行、Navicat、MySQL workbench来导入，推荐使用前两种方式。

​ 之后使用idea来生成Spring Boot，现在点击idea的文件–>新建项目，然后选择Spring Initialize

​ 然后按照图示创建项目就可以了

创建Spring Boot后的src（source）源代码文件夹如图所示：

一般src中写代码的顺序为：

1.model层，模型层，该层中每一个类对应数据库中的一个表

2.mapper层，数据访问层，也被称为DAO(Data Access Object)层，该层中都是接口，一个接口对应一个model，用来操作model。接口中的方法一般有对应数据库的SELECT，DELETE，INSERT，UPDATE方法，就是普通的增删查改，俗称CRUD。该层可以说是Spring中IoC最直观的体现，只需定义接口，无需提供具体的实现类，Spring中的IoC会自动为其注入依赖。具体对Spring中Ioc、DI、Aop等的理解可以参考《spring揭秘》这本书，在此不予多做讨论。

3.web层，也被称为Controller（控制层），用于提供前端访问后端的接口（这里的接口是指URL）。一般写好了程序也可用Controller来测试，不过更常用的方法是写单元测试的代码，这样方便快捷，不需要每次都一个个接口地测试。

至于enums文件夹只是我的项目特有的文件夹，就只有一个枚举类。SprinbootMybatisXmlApplication类是项目的配置和启动类，Spring Boot中还有许多可能令你莫名其妙的注解，具体注解的作用和使用方式可以参考Spring Boot官方文档 https://www.breakyizhan.com/springboot/3091.html 。

接下来再来看看resource（资源）文件夹

mybatis-config.xml是mybatis的配置类，里面主要定义了类型的别名。UserMapper.xml是对应src文件夹的mapper层的UserMapper接口，用来提供该接口中每个方法的xml（也就是SQL语句）的实现。application.properties是应用的配置类，主要有spring和mybatis的相关配置。

在src文件夹和resource文件夹还有值得我们关注的pom.xml文件

该文件主要作用是配置Spring Boot的相关依赖，一般我们如果在创建Spring Boot的时候勾选了web，该文件在本次项目中就可以不予理会了。不过如果没勾选web也不急，我们可以在dependency中添加web依赖

一般写代码的文件顺序为pom.xml–>application.properties–>model–>mapper–>mybatis.mapper–>web，按上面的顺序照着github上的源代码撸一下就大功告成了。

小技巧：使用win+左右方向键可以快速在windows上分屏，然后就可以边看源码边撸代码了

最后启动SpringbootXmlApplication类就可以在浏览器按着localhost:8080/getUsers等URL对每个方法就行测试了。

URL对应RequestMapping注解中的字符串而不是下面的方法名称哦。

当然如果你并不想一个个地用URL测试也可以编写单元测试，照着源代码中的test文件夹撸代码然后在对应测试类文件夹中运行该类就可以了。
